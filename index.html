The Nearest Special Unitary Matrix

Given a complex square matrix, how do we find its nearest special unitary matrix? Unlike the orthogonal, special orthogonal, and unitary cases, there is no simple SVD-based formula. Instead, the problem requires a more involved optimization over phases on the unit circle. This page explains the problem, common pitfalls, and the optimal solution.

The problem

Matrix nearness problems aim to find the closest member of a constrained set \(\mathcal S\) to given a matrix \(M\). Here the set is the special unitary group \(SU(n)\), and the distance is the Frobenius norm:

$$\min_{Q \in SU(n)} \|M - Q\|_F^2, \qquad SU(n) = \{\,Q \in \mathbb{C}^{n\times n} : Q^H Q = I,\ \det (Q) = 1\,\}$$

This projection is needed whenever a computation or algorithm produces a matrix that should be special unitary but isn't quite. For example, repeated floating-point multiplications can cause numerical drift away from \(SU(n)\), or a learning algorithm may return an estimate that doesn't exactly lie in the group. In both cases, we would wish to "correct" the matrix to the nearest valid matrix.

Why it's harder than O(n), U(n), and SO(n)

The classical analogues of this problem all have known one-line solutions built from the SVD (\(M = U\Sigma V^H\) where \(\Sigma = \mathrm{diag}(\sigma_1,\dots,\sigma_n)\) with \(\sigma_1\geq \dots \geq\sigma_n)\). Although the special unitary solution takes a similar form, it additionally requires a nontrivial optimization over a set of variables \(\{z_i\}\).

In the real case, \(\det(Q) = \pm 1\), so enforcing \(\det(Q) = 1\) means at worst flipping one singular direction, and the best choice is the one with the smallest singular value. In the complex case, however, the determinant of a unitary matrix can be any point on the unit circle (\(\det(Q) = e^{i\theta}\)), and the corrections to the singular directions take on a continuous set of values rather than a discrete sign flip. With this additional flexibility, the optimal total correction is distributed among all singular directions in a coupled way.

Two suboptimal heuristics

Here are two commonly used naive approaches to correct the matrix. Although the methods yield valid \(SU(n)\) matrices, they do not necessarily produce the nearest one.

Naive I — global phase correction

Take the nearest unitary matrix \(A = UV^H\) and scale it so that its determinant becomes one: \(Q = \det(A)^{-1/n} A\). This spreads the phase correction evenly across all directions.

Naive II — \(SO(n)\) Analogue

This is the exact analogue of the special orthogonal solution where the correction is applied entirely to the smallest singular value direction: \(Q = U\,\mathrm{diag}(1,\dots,1,\rho^{*})\,V^H\).

The true solution lies in between: the phase correction should be shared across directions, but not necessarily evenly.

The solution: distributing a phase budget

Properly minimizing the Frobenius distance reveals that the optimal \(Q\) always has the form:

$$Q = U Z^{*} V^{H}, \qquad Z = \mathrm{diag}(z_1,\dots,z_n),\quad |z_i| = 1$$

and the matrix problem collapses to choosing \(n\) unit-modulus phases that maximize the constrained objective:

$$\max_{|z_i|=1}\ 2\sum_{i=1}^{n}\sigma_i\,\Re(z_i)\quad \text{s.t.}\quad \prod_{i=1}^{n} z_i = \rho := \det(U)\det(V^{H})$$

Think of \(n\) knobs on the unit circle. Each knob wants to sit at angle zero (where \(\Re(z_i)\) is maximal), but their angles must sum to \(\arg\rho\), a fixed phase budget that has to be distributed somehow. Turning the knob attached to a large \(\sigma_i\) is expensive, while turning one attached to a small \(\sigma_i\) is relatively cheap.

The stationarity conditions specify the distribution rule exactly. Writing \(a_i = \Re(z_i),b_i=\Im(z_i)\), every optimum satisfies:

$$\sigma_i\, b_i = \sigma_j\, b_j \quad \text{for all } i,j$$

Thus, the imaginary parts are all related to the ratio of singular values. This provides enough constraints to solve for the optimal solution.

Solving for the phases

From the constraints, we can find the exact solution by eliminating variables and solving for the roots of a univariate polynomial. However, the polynomial's degree grows like \(2^{n-1}(n-1)\), making this approach numerically fragile past \(n = 4\). Alternatively, since a single scalar parameterizes all variables through our constraints, we can perform a one-dimensional root search on a bounded interval instead which avoids constructing the polynomial directly. This numerical approach is robust even for large \(n\).

Special case n = 2: This case admits a closed-form formula via the \(SU(2)\) ↔ quaternion isomorphism and does not require SVD:

$$Q \;=\; \frac{M + \mathrm{adj}(M)^{H}}{\sqrt{\;\|M\|_F^2 + 2\,\Re(\det (M))\;}}$$

Example: n = 2

Take:

$$M = \begin{pmatrix} 1+i & 0.5\,i \\ -0.5 & 1-0.5\,i \end{pmatrix}, \qquad \det (M) = 1.5 + 0.75\,i,\qquad \|M\|_F^2 = 3.75$$

The formula needs only the adjugate and two scalars: \(\;\|M\|_F^2 + 2\,\Re(\det (M)) = 3.75 + 3 = 6.75\), so the denominator is \(\sqrt{6.75} \approx 2.598\), and:

$$M + \mathrm{adj}(M)^H = \begin{pmatrix} 2+1.5\,i & 0.5+0.5\,i \\ -0.5+0.5\,i & 2-1.5\,i \end{pmatrix} \;\;\Longrightarrow\;\; Q \approx \begin{pmatrix} 0.770+0.577\,i & 0.192+0.192\,i \\ -0.192+0.192\,i & 0.770-0.577\,i \end{pmatrix}$$

One can verify \(Q^H Q = I\) and \(\det (Q) = 1\) to machine precision. The table below compares the methods.

import numpy as np
from nearest_su import su2_algebraic, naive_1, naive_2

np.set_printoptions(precision=17, linewidth=200)

M = np.array([[1 + 1j, 0.5j],
              [-0.5,   1 - 0.5j]], dtype=complex)
U, s, Vh = np.linalg.svd(M)

Q = su2_algebraic(M)
print(naive_1(U, s, Vh)); print(naive_2(U, s, Vh)); print(Q)
print("dist naive I  = %.17g" % np.linalg.norm(M - naive_1(U, s, Vh)))
print("dist naive II = %.17g" % np.linalg.norm(M - naive_2(U, s, Vh)))
print("dist optimal      = %.17g" % np.linalg.norm(M - Q))

Example: n = 4

Now take a generic 4x4 complex matrix (see code below for exact values). Its SVD gives:

$$\sigma \approx (4.151,\; 2.197,\; 1.357,\; 0.910), \qquad \rho \approx -0.658 + 0.753\,i,\quad \arg\rho \approx 131.2^{\circ}$$

So the four phases must turn a combined \(\approx 131.2^{\circ}\). The optimal solution distributes that budget to each phase direction according to its singular value as seen below.

import numpy as np
from nearest_su import numerical_optimization, naive_1, naive_2

np.set_printoptions(precision=17, linewidth=200)

rng = np.random.default_rng(25700)
M = rng.uniform(-2, 2, (4, 4)) + 1j * rng.uniform(-2, 2, (4, 4))
U, s, Vh = np.linalg.svd(M)

Q = numerical_optimization(U, s, Vh)
z = np.conjugate(np.diag(U.conj().T @ Q @ Vh.conj().T))   # optimal phases
print(M); print(s); print(z)
print(naive_1(U, s, Vh)); print(naive_2(U, s, Vh)); print(Q)
print("dist naive I  = %.17g" % np.linalg.norm(M - naive_1(U, s, Vh)))
print("dist naive II = %.17g" % np.linalg.norm(M - naive_2(U, s, Vh)))
